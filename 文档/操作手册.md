# ASR合板工具 — 操作手册

> 版本: v0.8.0  
> 更新日期: 2026-04-18

---

## 1. 产品简介

ASR合板工具是一款基于 **SenseVoice ASR** 语音识别引擎的影视后期辅助工具。它能自动将拍摄素材（视频+音频）进行语音识别、字幕匹配，并生成可直接导入剪辑软件的时间线文件。

### 核心功能

- 批量 ASR 语音识别（支持 GPU/CUDA 加速）
- 自动字幕与视频匹配
- 多格式时间线导出（XML / FCPXML / SRT）
- 拖放文件导入
- ASR 断点续传
- 中英文双语界面
- 深色/浅色主题切换

### 技术环境要求

| 组件 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 (x64) |
| FFmpeg | 已安装，用于音视频处理 |
| sherpa-onnx | 已安装，含 SenseVoice 模型 |
| GPU（可选） | NVIDIA CUDA 支持，加速 ASR 识别 |

## 2. 首次配置

### 2.1 启动应用

双击 `asr_tools.exe` 启动应用。

### 2.2 设置外部工具路径

1. 点击左侧导航栏 **「设置」**
2. 配置以下路径：

| 设置项 | 说明 | 示例路径 |
|--------|------|----------|
| FFmpeg 目录 | FFmpeg 安装目录（含 ffmpeg.exe） | `G:\data\app\DIT\ffmpeg` |
| sherpa-onnx 目录 | sherpa-onnx 安装目录 | `G:\data\app\DIT\sherpa-onnx` |
| ASR 模型目录 | SenseVoice 模型存放目录（可选） | 留空使用默认 |
| 代理地址 | 网络代理（用于下载依赖） | `192.168.0.211:7890` |

3. 修改后自动保存

### 2.3 界面语言与主题

- **语言切换**：设置 → 界面语言 → 简体中文 / English
- **主题切换**：设置 → 主题 → 深色主题 / 浅色主题

## 3. 工作流程

应用采用 **四步工作流**，从工程创建到时间线导出：

```
创建工程 → 素材导入 → ASR识别 → 字幕匹配 → 时间线生成与导出
```

### 3.1 第一步：创建工程

1. 在 **工程列表** 页面，点击右下角 **「新建工程」** 按钮
2. 输入工程名称（建议格式：项目名_拍摄日期）
3. 点击 **「创建」**

> 工程创建后自动进入工程详情页面。

### 3.2 第二步：素材导入

进入 **「素材导入」** 步骤，需要分别导入视频和音频素材。

#### 方式一：目录选择（推荐）

1. 在 **「视频目录」** 栏，点击 **「选择目录」** 按钮
2. 选择包含视频文件的目录
3. 应用自动扫描目录中的视频文件（mp4, mov, avi, mkv, mxf, wmv, flv, webm）
4. 在 **「音频目录」** 栏重复以上操作，选择包含音频的目录（wav, mp3, aac, flac, ogg, wma, m4a）
5. 确认视频和音频文件列表无误后，点击 **「确认导入，进入下一步」**

#### 方式二：拖放导入

1. 直接从文件资源管理器拖放视频/音频文件到应用窗口
2. 文件自动根据扩展名分类为视频或音频
3. 已存在的文件会自动跳过（去重）

#### 文件列表说明

| 信息 | 说明 |
|------|------|
| 文件图标 | 🎬 视频 / 🎵 音频 |
| 文件名 | 显示文件名和扩展名 |
| 时长标签 | 蓝色标签显示文件时长（mm:ss 格式） |
| 文件计数 | 显示当前栏目的文件总数 |

> 提示：可以随时点击 **「更换目录」** 重新选择，已有文件会被替换。

### 3.3 第三步：ASR 语音识别

进入 **「ASR识别」** 步骤。

#### 环境检测

页面顶部自动显示 sherpa-onnx 环境状态：
- **GPU 加速**（绿色）：检测到 CUDA，识别速度更快
- **CPU 模式**（橙色）：使用 CPU 识别，速度较慢
- **VAD+ASR**（蓝色）：支持语音活动检测联合识别
- **环境异常**（黄色）：sherpa-onnx 路径配置有误

#### 开始识别

1. 确认环境状态正常后，点击 **「开始识别」**
2. 观察识别进度：
   - **总体进度条**：显示已完成/总文件数
   - **单文件进度**：每个文件独立显示状态
     - ⏳ 待识别
     - 🔊 提取音频
     - 🎤 识别中
     - 💾 保存中
     - ✅ 已完成
     - ⏭️ 已跳过（已有识别结果）
     - ❌ 失败

#### 识别中断与续传

- **取消识别**：点击 **「取消」** 按钮中断当前识别
- **继续识别**：中断后点击 **「继续识别」** 从断点恢复
  - 已完成的文件自动跳过
  - 只对未完成的文件重新识别
- **跨会话续传**：即使关闭应用重新打开，已识别的结果保存在数据库中，再次点击「开始识别」也会自动跳过

#### 重试失败文件

- 识别失败的文件会显示 **「重试」** 按钮
- 点击可单独重新识别该文件

#### ASR 配置（设置页面）

| 配置项 | 选项 | 说明 |
|--------|------|------|
| VAD 模式 | 标准模式 / 长音频模式 | 长音频模式适合连续录音 |
| 识别语言 | 自动检测 / 中文 / 英文 / 日文 / 韩文 / 粤语 | 自动检测覆盖所有语言 |

#### 识别完成

- 全部完成后显示绿色 **「确认识别完成，进入匹配」** 按钮
- 部分成功也可继续（失败的文件可以后续重试）

### 3.4 第四步：字幕匹配

进入 **「字幕匹配」** 步骤。

#### 开始匹配

1. 点击 **「开始匹配」**
2. 系统自动将 ASR 识别的字幕与视频文件进行匹配
3. 匹配结果按置信度分级：
   - 🟢 **高置信度**（≥90%）：字幕与视频高度匹配
   - 🟡 **中置信度**（70%-90%）：部分匹配，建议人工确认
   - 🔴 **低置信度**（<70%）：匹配不确定
   - ⚫ **无匹配**：未找到对应关系

#### 字幕对比

- 点击匹配结果可展开 **字幕对比** 面板
- 左侧显示原始 ASR 字幕，右侧显示匹配结果
- 方便快速确认匹配准确性

#### 确认匹配

- 确认匹配结果后，点击 **「确认匹配完成，进入时间线」**

### 3.5 第五步：时间线生成与导出

进入 **「时间线」** 步骤。

#### 生成时间线

1. 点击 **「生成时间线」**
2. 系统根据匹配结果自动生成时间线数据
3. 时间线预览显示所有片段（视频+字幕对齐）

#### 导出格式

| 格式 | 说明 | 适用软件 |
|------|------|----------|
| XML | Adobe Premiere Pro | Premiere Pro, DaVinci Resolve |
| FCPXML | Final Cut Pro XML | Final Cut Pro X |
| SRT | 通用字幕格式 | 所有播放器、剪辑软件 |
| 批量 SRT | 按视频文件名分别导出 SRT | 批量字幕交付 |

#### 导出操作

- 点击对应的 **导出按钮**（XML / FCPXML / SRT / 批量 SRT）
- 选择保存位置
- 导出完成后显示成功提示

## 4. 工程管理

### 4.1 工程列表

- 首页显示所有已创建的工程
- 每个工程卡片显示：名称、状态、视频/音频文件数、创建时间

### 4.2 搜索与排序

- **搜索**：点击搜索图标展开搜索框，输入关键字搜索工程名称
- **排序**：点击排序图标选择排序方式
  - 更新时间 新→旧
  - 更新时间 旧→新
  - 名称 A→Z
  - 创建时间 新→旧

### 4.3 工程状态

| 状态 | 含义 |
|------|------|
| 已创建 | 工程刚创建，等待导入素材 |
| 已导入 | 素材导入完成，等待 ASR 识别 |
| 识别中 | ASR 识别正在进行 |
| 已识别 | ASR 识别完成，等待字幕匹配 |
| 已匹配 | 字幕匹配完成，可以生成时间线 |
| 已生成 | 时间线已生成，可以导出 |

### 4.4 删除工程

- 在工程卡片上右键或点击菜单 → **「删除」**
- 确认删除后不可恢复

## 7. 常见问题

### Q: 启动后环境检测显示异常？
**A:** 进入设置页面检查 FFmpeg 和 sherpa-onnx 路径是否正确。确保路径下有对应的可执行文件。

### Q: ASR 识别速度很慢？
**A:** 
- 检查是否启用了 GPU 加速（环境指示器应显示「GPU 加速」）
- 确认安装了 CUDA 和对应版本的 NVIDIA 驱动
- 使用「长音频模式」减少分段次数

### Q: 识别中途中断了怎么办？
**A:** 直接再次打开工程，点击「继续识别」，系统会自动跳过已完成的文件。

### Q: 导出的 SRT 字幕在播放器中乱码？
**A:** SRT 文件默认使用 UTF-8 编码，请确认播放器支持 UTF-8。

### Q: 窗口位置每次启动都居中？
**A:** v0.8.0 已支持窗口位置记忆。首次使用时移动窗口到合适位置，下次启动会自动恢复。
