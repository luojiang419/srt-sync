## 2. 数据模型设计

### 2.1 AsrProject（工程）

```dart
class AsrProject {
  String id;              // UUID
  String name;            // 工程名称
  String? videoDirectory; // 视频目录路径
  String? audioDirectory; // 音频目录路径
  ProjectStatus status;   // 工程状态枚举
  String asrModel;        // ASR 模型名称（默认 sense-voice-small）
  DateTime createdAt;     // 创建时间
  DateTime updatedAt;     // 更新时间
}
```

**状态枚举 `ProjectStatus`：**

```
created → imported → recognizing → recognized → matched → timeline → completed
```

### 2.2 MediaFile（媒体文件）

```dart
class MediaFile {
  String id;                // UUID
  String projectId;         // 所属工程 ID
  String filePath;          // 文件完整路径
  MediaType type;           // video | audio
  int? durationMs;          // 时长（毫秒），由 FFmpeg 提取
  SubtitleStatus subtitleStatus; // 字幕状态：pending/completed/failed
  DateTime createdAt;       // 创建时间
}
```

### 2.3 SubtitleClip（字幕片段）

```dart
class SubtitleClip {
  String id;            // UUID
  String mediaFileId;   // 所属媒体文件 ID
  int startMs;          // 开始时间（毫秒）
  int endMs;            // 结束时间（毫秒）
  String text;          // 字幕文本
  int sortOrder;        // 排序序号
}
```

### 2.4 MatchPair（匹配对）

```dart
class MatchPair {
  String id;             // UUID
  String projectId;      // 所属工程 ID
  String videoFileId;    // 视频文件 ID
  String audioFileId;    // 音频文件 ID
  double confidence;     // 匹配置信度（0.0~1.0）
  bool isConfirmed;      // 是否人工确认
  int sortOrder;         // 排序序号
}
```

### 2.5 TimelineData（时间线数据）

```dart
class TimelineData {
  String id;                  // UUID
  String projectId;           // 所属工程 ID
  String videoFilePath;       // 视频文件路径
  String? audioFilePath;      // 对齐音频路径
  List<SubtitleClip> subtitles; // 关联字幕
  int startTimeMs;            // 开始时间
  int durationMs;             // 持续时长
}
```

### 2.6 数据库表关系

```
AsrProject (1) ──< (N) MediaFile
MediaFile (1) ──< (N) SubtitleClip
AsrProject (1) ──< (N) MatchPair
MatchPair (N) >── (1) MediaFile [video]
MatchPair (N) >── (1) MediaFile [audio]
AsrProject (1) ──< (N) TimelineData
```

## 3. 核心业务逻辑

### 3.1 素材导入流程

```
用户选择目录/拖放文件
        │
        ▼
MediaScanService.scanDirectory(path, type)
  ├─ 遍历目录，按扩展名过滤
  ├─ 排序（文件名 A→Z）
  └─ 返回 ScannedFile 列表
        │
        ▼
FfmpegService.getDuration(filePath)
  ├─ 调用 ffprobe 获取时长
  └─ 返回毫秒数
        │
        ▼
DatabaseService.insertMediaFile(mediaFile)
  ├─ 生成 UUID
  └─ 写入 media_files 表
        │
        ▼
更新 AsrProject 的 videoDirectory/audioDirectory
```

**支持的文件格式：**
- 视频：`.mp4 .mov .avi .mkv .mxf .wmv .flv .webm`
- 音频：`.wav .mp3 .aac .flac .ogg .wma .m4a`

### 3.2 ASR 语音识别流程

```
AsrBatchService.batchRecognize()
        │
        ▼
SherpaOnnxService.checkEnv(sherpaOnnxPath, modelName)
  ├─ 检查 sherpa-onnx 二进制文件
  ├─ 检查模型文件
  ├─ 检测 GPU/CUDA 可用性
  └─ 返回 SherpaOnnxEnv
        │
        ▼
对每个音频文件循环：
        │
        ├─ 检查数据库已有字幕 → 跳过已完成文件（断点续传核心）
        │
        ├─ FfmpegService.extractWav(inputPath, wavPath)
        │   └─ ffmpeg -i input -ar 16000 -ac 1 output.wav
        │
        ├─ SherpaOnnxService.recognize(wavPath, env, vadPreset)
        │   ├─ 构建命令行参数
        │   │   ├─ --vad-threshold / --vad-min-silence-duration / ...
        │   │   ├─ GPU: --provider=cuda
        │   │   └─ CPU: 不添加 --provider 参数
        │   ├─ 执行外部进程 (process_run)
        │   ├─ 解析 JSON 输出
        │   └─ 返回 List<AsrSegment> [{startTime, endTime, text}]
        │
        ├─ 保存到数据库
        │   ├─ 清除旧字幕
        │   ├─ 创建 SubtitleClip 列表
        │   └─ 批量插入
        │
        └─ 更新 MediaFile 的 subtitleStatus
```

**GPU/CUDA 自动降级机制：**
1. 首先尝试 `--provider=cuda` 运行
2. 如果 CUDA 不可用（进程异常退出），自动移除 `--provider` 参数
3. 使用 CPU 模式重新运行

**VAD 参数预设：**

| 参数 | 标准 | 长音频 |
|------|------|--------|
| threshold | 0.5 | 0.4 |
| minSilenceDuration | 0.25s | 0.5s |
| minSpeechDuration | 0.25s | 0.25s |
| maxSpeechDuration | 5.0s | 30.0s |

### 3.3 字幕匹配算法

```
SubtitleMatchService.matchProject(projectId)
        │
        ▼
加载所有视频文件和音频文件
加载所有音频文件的 ASR 字幕
        │
        ▼
对每个视频文件：
        │
        ├─ 提取视频文件名（去掉扩展名）
        │
        ├─ 滑动窗口匹配（窗口大小 = 3）
        │   ├─ 计算文件名与 ASR 字幕文本的相似度
        │   │   └─ 使用 Levenshtein 距离或包含关系
        │   ├─ 窗口内取最高相似度
        │   └─ 相似度 ≥ 0.85 视为匹配
        │
        ├─ 计算置信度
        │   ├─ ≥ 0.9 → 高
        │   ├─ ≥ 0.7 → 中
        │   └─ < 0.7 → 低
        │
        └─ 保存 MatchPair 到数据库
```

**匹配算法关键参数：**
- `matchWindowSize = 3`：上下文窗口大小
- `matchSimilarityThreshold = 0.85`：匹配阈值
- `matchConfidenceHigh = 0.9`：高置信度分界
- `matchConfidenceMedium = 0.7`：中置信度分界

### 3.4 时间线生成与导出

```
TimelineNotifier.buildTimeline(projectId)
        │
        ▼
加载所有 MatchPair
对每个匹配对：
        │
        ├─ AudioAlignService.alignAudio(videoPath, audioPath, subtitles)
        │   ├─ FFmpeg 分析音频波形
        │   ├─ 计算音频对齐偏移量
        │   └─ 返回对齐参数（offset、duration）
        │
        └─ 生成 TimelineData
            ├─ 视频路径
            ├─ 音频路径
            ├─ 字幕列表
            └─ 起止时间

导出格式：
├─ ExportService → XML (Premiere)
├─ ExportService → FCPXML (Final Cut Pro)
└─ ExportService → SRT（通用字幕）
```

## 6. 外部进程调用设计

### 6.1 FFmpeg 调用

| 操作 | 命令 | 用途 |
|------|------|------|
| 获取时长 | `ffprobe -v quiet -show_entries format=duration -of csv=p=0 <file>` | 素材导入时获取时长 |
| 提取 WAV | `ffmpeg -i <input> -ar 16000 -ac 1 -y <output.wav>` | ASR 前提取音频 |
| 裁切音频 | `ffmpeg -i <input> -ss <start> -t <duration> -y <output>` | 时间线裁切 |

### 6.2 sherpa-onnx 调用

```
sherpa-onnx-offline \
  --tokens=<model_dir>/tokens.txt \
  --sense-voice-model=<model_dir>/model.int8.onnx \
  --vad-model=<vad_dir>/silero_vad.onnx \
  --wav=<input.wav> \
  --vad-threshold=<threshold> \
  --vad-min-silence-duration=<duration> \
  --vad-max-speech-duration=<duration> \
  [--provider=cuda]          ← GPU 可用时添加
  --use-vad-with-asr=1
```

**输出格式（JSON）：**
```json
{
  "segments": [
    {"start": 0.0, "end": 2.5, "text": "识别的文字内容"},
    ...
  ]
}
```

## 7. 数据持久化设计

### 7.1 SQLite 数据库

数据库名：`asr_tools.db`，版本：1

**表结构：**

```sql
-- 工程表
CREATE TABLE projects (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  video_directory TEXT,
  audio_directory TEXT,
  status TEXT NOT NULL DEFAULT 'created',
  asr_model TEXT NOT NULL DEFAULT 'sense-voice-small',
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);

-- 媒体文件表
CREATE TABLE media_files (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  file_path TEXT NOT NULL,
  type TEXT NOT NULL,           -- 'video' | 'audio'
  duration_ms INTEGER,
  subtitle_status TEXT DEFAULT 'pending',
  created_at TEXT NOT NULL,
  FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);

-- 字幕片段表
CREATE TABLE subtitle_clips (
  id TEXT PRIMARY KEY,
  media_file_id TEXT NOT NULL,
  start_ms INTEGER NOT NULL,
  end_ms INTEGER NOT NULL,
  text TEXT NOT NULL,
  sort_order INTEGER NOT NULL,
  FOREIGN KEY (media_file_id) REFERENCES media_files(id) ON DELETE CASCADE
);

-- 匹配对表
CREATE TABLE match_pairs (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  video_file_id TEXT NOT NULL,
  audio_file_id TEXT NOT NULL,
  confidence REAL NOT NULL,
  is_confirmed INTEGER DEFAULT 0,
  sort_order INTEGER NOT NULL,
  FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
  FOREIGN KEY (video_file_id) REFERENCES media_files(id),
  FOREIGN KEY (audio_file_id) REFERENCES media_files(id)
);

-- 时间线数据表
CREATE TABLE timeline_data (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  video_file_path TEXT NOT NULL,
  audio_file_path TEXT,
  start_time_ms INTEGER NOT NULL,
  duration_ms INTEGER NOT NULL,
  FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);
```

## 8. 特殊机制设计

### 8.1 窗口位置记忆

```
main.dart（启动时）
  ├─ 预加载 settings JSON（不依赖 Riverpod）
  ├─ 检查 hasWindowGeometry
  ├─ 有 → 恢复窗口位置和大小
  └─ 无 → 默认 1280×800 居中

WindowGeometryListener（运行时）
  ├─ 实现 WindowListener mixin
  ├─ 监听 onWindowMoved / onWindowResized
  ├─ 500ms 防抖
  └─ 直接写入 settings JSON（独立于 Riverpod）
```

### 8.2 ASR 断点续传

```
核心机制：batchRecognize 内部跳过已有字幕的文件

startBatchRecognize：
  for file in audioFiles:
    if getSubtitleClips(file.id).isNotEmpty → skip

resumeRecognize（续传入口）：
  1. 重建进度列表（从数据库读取已完成文件的字幕）
  2. 调用 batchRecognize（自动跳过已完成文件）

跨会话续传：
  由于识别结果保存在数据库中，关闭应用后重新打开
  点击「开始识别」也会自动跳过已完成文件
```

### 8.3 国际化系统

```
AppLocalizations
  ├─ Map<String, Map<String, String>> 翻译映射
  ├─ t(key) → 无参翻译
  └─ LocalizationsX 扩展：
      ├─ context.loc.t('key')
      └─ context.locp('key', {'var': value}) → 带参数替换

切换流程：
  settings.locale = 'en'
      │
      ▼
  app.dart 监听 settingsProvider
      │
      ▼
  MaterialApp.router(locale: Locale('en', 'US'))
      │
      ▼
  所有 context.loc.t() 调用返回英文
```

### 8.4 主题系统

```
AppTheme
  ├─ darkTheme  → 深色 ThemeData（#1A1A2E 系列）
  └─ lightTheme → 浅色 ThemeData（#F5F5FA 系列）

切换流程：
  settings.themeMode = 'light'
      │
      ▼
  app.dart 监听 settingsProvider
      │
      ▼
  MaterialApp.router(theme: AppTheme.lightTheme)
      │

注意：AppTheme 中定义的静态颜色常量（如 AppTheme.highlight）
  被 UI 组件直接引用，不随主题变化。
  如需完全动态化，需改为从 Theme.of(context) 读取。
```

### 9.3 添加新 ASR 引擎

实现新的 Service 类（类似 `SherpaOnnxService`），在 `AsrBatchService` 中替换引擎调用即可。
