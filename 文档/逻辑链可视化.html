🎤 ASR 识别流程

graph TD
    START[开始批量识别] --> ENV[检查 sherpa-onnx 环境]
    ENV --> |环境异常| ERR[返回错误]
    ENV --> |环境就绪| LOOP{遍历音频文件}

    LOOP --> |检查已有字幕| SKIP{已有字幕?}
    SKIP --> |是| SKIP_DONE[跳过 ✅ 断点续传核心]
    SKIP --> |否| EXTRACT[FFmpeg 提取 16kHz WAV]

    EXTRACT --> GPU_CHECK{检测 GPU}
    GPU_CHECK --> |CUDA 可用| GPU_RUN[CUDA 模式识别]
    GPU_CHECK --> |仅 CPU| CPU_RUN[CPU 模式识别]

    GPU_RUN --> |CUDA 失败| AUTO_FALLBACK[自动降级到 CPU]
    GPU_RUN --> |成功| PARSE
    CPU_RUN --> PARSE[解析 JSON 输出]
    AUTO_FALLBACK --> PARSE

    PARSE --> SPLIT{文本 > 42字?}
    SPLIT --> |是| BY_PUNCT[按标点拆分
时间按字符等比分配]
    SPLIT --> |否| SAVE
    BY_PUNCT --> SAVE[保存到 subtitle_clips 表]
    SAVE --> LOOP
    SKIP_DONE --> LOOP

    LOOP --> |全部完成| RESULT[返回 AsrBatchResult]
    LOOP --> |用户取消| CANCEL[保留进度，可续传]

🔗 字幕匹配算法流程

graph TD
    START[开始匹配] --> LOAD_V[加载所有视频文件]
    LOAD_V --> LOAD_A[加载所有音频文件]
    LOAD_A --> LOAD_S[加载所有音频的 ASR 字幕]

    LOAD_S --> LOOP_V{遍历每个视频}
    LOOP_V --> NAME[提取视频文件名
去掉扩展名]

    NAME --> SCORE1[文件名相似度 × 0.6]
    NAME --> SCORE2[时长相似度 × 0.3]
    NAME --> SCORE3[文本自评分 × 0.1]

    SCORE1 --> COMBO[综合得分]
    SCORE2 --> COMBO
    SCORE3 --> COMBO

    COMBO --> RANK[所有候选按得分排序]
    RANK --> GREEDY[贪心分配:
已分配文件不再参与]
    GREEDY --> FILTER{得分 ≥ 0.3?}
    FILTER --> |否| DISCARD[丢弃该匹配]
    FILTER --> |是| SAVE[保存 MatchPair]

    SAVE --> CONF{置信度 ≥ 0.9?}
    CONF --> |是| AUTO[自动确认 ✓]
    CONF --> |否| MANUAL[待人工确认]

    AUTO --> LOOP_V
    MANUAL --> LOOP_V
    DISCARD --> LOOP_V
    LOOP_V --> |完成| END[返回匹配结果]

    style AUTO fill:#3fb950,stroke:#3fb950,color:#000

📐 三维评分体系

文件名相似度（权重 60%）

基于 Levenshtein 编辑距离
完全匹配 = 1.0
包含关系 = 0.9
模糊匹配 = 编辑距离比

时长相似度（权重 30%）

视频与音频的时长差异比
10% 以内 = 高度相似
线性衰减计算

文本自评分（权重 10%）

从视频文件名提取数字序列
检查是否出现在 ASR 字幕文本中
辅助验证匹配正确性

高置信度 ≥ 0.9（自动确认）
中置信度 0.7 ~ 0.9
低置信度 0.3 ~ 0.7
低于 0.3（丢弃）

📁 导出格式对比

格式适用软件内容时间精度
XML Adobe Premiere Pro 视频片段 + 音频对齐 + 字幕轨道 24fps 帧级
FCPXML Final Cut Pro X / DaVinci 完整项目结构 + 资源引用 2400 时间基
SRT 所有播放器/剪辑软件 序号 + 时间轴 + 字幕文本 毫秒级
裁切 WAV 音频后期 FFmpeg 精确裁切对齐音频 毫秒级
