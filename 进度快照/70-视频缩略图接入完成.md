# 进度快照 70 - 视频缩略图接入完成

## 版本信息
- 当前源码版本: `1.1.12+30`
- 阶段备份: `backup/v0.37.0`
- 发布状态: 待编译发布

## 已完成内容

### 1. 视频缩略图缓存链路已接入
- 已新增 `VideoThumbnailService`
- 缩略图缓存已统一落到现有 `data/projects/<projectId>/thumbnails/`
- 命名规则已固定为：
  - `<mediaFileId>.jpg`
- 默认抽取视频距开头约 1 秒的画面
- 短视频会回退到起始位置取帧

### 2. 媒体模型与数据库已支持缩略图字段
- `MediaFile` 已新增可空字段：
  - `thumbnailPath`
- `copyWith / fromMap / toMap` 已同步支持
- `media_files` 表已新增：
  - `thumbnail_path TEXT`
- 已把数据库版本提升到 `4`
- 旧库启动时会自动补列，不做破坏性迁移

### 3. 视频导入时已自动生成缩略图
- 新导入视频素材时，会在媒体探测后立即尝试生成缩略图
- 缩略图生成成功后，会把缓存路径直接写入数据库
- 音频素材不会生成缩略图
- 若 FFmpeg 不可用、源视频不存在或抽帧失败，会安全回退为无图状态，不阻塞导入

### 4. 当前项目历史视频已支持自动补齐缩略图
- 打开当前项目后，会自动检查本项目视频素材中缺失缩略图的条目
- 对缺图且源文件仍存在的视频，会在后台自动补生成
- 生成完成后会刷新当前项目状态，使导入页和一键合板页都能看到缩略图
- 仅补齐当前项目，不扫描全部项目

### 5. 两个页面的卡片已接入缩略图显示
- `素材导入` 页中：
  - 视频素材卡片左侧已显示固定比例缩略图
  - 音频素材卡片保持现状
- `一键合板` 页中：
  - 匹配结果卡片左侧已显示视频缩略图
  - 原有文件名、状态、音频命中、置信度和复核信息保留
- 无图时会显示稳定占位块，不会挤压现有按钮布局

### 6. 删除工程与重导视频的缓存清理已接入
- 删除工程时，会同步清理该工程在 `data/projects/<projectId>` 下的缩略图缓存
- 重新导入视频目录前，会先清空当前工程缩略图缓存，再生成新一轮缩略图

### 7. 自动化验证已完成
- `flutter analyze` 针对缩略图相关文件
  - 通过，无问题
- `flutter test --no-pub test/video_thumbnail_service_test.dart test/video_thumbnail_ui_test.dart test/widget_test.dart`
  - 通过
- `flutter test --no-pub`
  - 通过
- `flutter analyze lib test`
  - 无新增 error / warning
  - 仍存在仓库既有 info 级提示
