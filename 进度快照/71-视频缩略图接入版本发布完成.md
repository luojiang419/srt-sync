# 进度快照 71 - 视频缩略图接入版本发布完成

## 版本信息
- 新版本号: `v1.1.12`
- `pubspec.yaml`: `1.1.12+30`
- 阶段备份: `backup/v0.37.0`
- 发布目录: `dist/v1.1.12`

## 已完成内容

### 1. 视频缩略图缓存体系已接入
- 已新增 `VideoThumbnailService`
- 视频缩略图统一缓存到：
  - `data/projects/<projectId>/thumbnails/<mediaFileId>.jpg`
- 抽帧策略已固定为距开头约 1 秒
- 短视频自动回退到起始位置取帧
- 输出格式为 JPG，小图尺寸统一，适合列表卡片显示

### 2. 媒体模型与数据库已支持缩略图持久化
- `MediaFile` 已新增：
  - `thumbnailPath`
- `media_files` 表已新增：
  - `thumbnail_path`
- 数据库版本已提升到 `4`
- 旧库启动时会自动补列，不影响已有工程读取

### 3. 视频导入和当前项目回填都已接入
- 新导入视频素材时会立即尝试生成缩略图并写入数据库
- 当前项目中历史缺图视频会在项目加载后自动后台补齐
- 仅处理当前项目，不扫描所有项目
- FFmpeg 不可用、源文件不存在或抽帧失败时，会回退为无图占位，不阻塞页面和导入流程

### 4. 两个目标页面已显示视频缩略图
- `素材导入` 页：
  - 视频素材卡片左侧已显示缩略图
  - 音频素材卡片保持原样
- `一键合板` 页：
  - 匹配结果卡片左侧已显示视频缩略图
  - 原有状态、复核、音频命中、置信度信息已保留
- 无图时会显示稳定占位块，不会挤压现有布局

### 5. 工程级缓存清理已接入
- 删除工程时，会同步删除 `data/projects/<projectId>` 下的缩略图缓存
- 重新导入视频目录前，会先清空当前工程旧缩略图缓存，再生成新图

### 6. 构建与启动冒烟已完成
- `flutter build windows --release`
  - 通过
- 已生成：
  - `build/windows/x64/runner/Release/asr_tools.exe`

- 已完成启动冒烟：
  - `dist/v1.1.12/asr_tools.exe`
  - 正常拉起并完成自动关闭验证
- 启动后已确认存在：
  - `dist/v1.1.12/data/config/asr_tools_settings.json`
  - `dist/v1.1.12/data/database/asr_tools.db`
  - `dist/v1.1.12/data/projects`
  - `dist/v1.1.12/data/temp`
  - `dist/v1.1.12/data/projects/<projectId>/thumbnails`
